## Objective
Lock the default toolset to least privilege (computer_use/browser off) and sandbox execution (docker).

## Environment note
This test host had **no container runtime**. Per an explicit decision, **Colima** + the docker CLI were installed on the test host (host dev tooling only — the live `~/.hermes` install was untouched) so `terminal.backend: docker` could be verified for real. Image `nikolaik/python-nodejs:python3.11-nodejs20` pulled (2.21GB).

## What was tested
1. computer_use + browser unavailable in the gov profile.
2. `terminal.backend` resolves to docker and bridges to `TERMINAL_ENV=docker`.
3. A sandboxed shell command runs via the docker backend.
4. A normal coding task still works.

## Results (actual)
- **Toolset:** baseline resolved toolset includes `browser` + `computer_use`; gov profile **removes both** (none present), while `terminal`/`file` remain. `tool_denied('computer_use')=True`, `tool_denied('browser')=True`, `tool_denied('file_read')=False`.
- **Config + bridge:** `terminal.backend=docker`; `apply_terminal_config_to_env` → `TERMINAL_ENV=docker`.
- **Sandbox proof:** host `uname -s` = `Darwin`; agent's sandboxed shell `uname -s` = **`Linux`** → the command ran inside the container, not the host (LG-CH-05).
- **Coding task:** agent created `calc.py` and computed `add(2,3)=5` in the sandbox.

## Expected-deny / expected-block cases
- `computer_use`, `browser` toolsets → **unavailable** in the gov profile (LG-AZ-11/03).

## Capability preservation
Shell stays available but **sandboxed** + approval-gated (mission-required for a coding agent); coding works in-sandbox. Note: `mount_docker_cwd` defaults false, so agent edits live in-container — real deployments must set `TERMINAL_DOCKER_MOUNT_CWD_TO_WORKSPACE=true` + mount the safe-root for persistent host edits (recorded in `regression-log.md`).

## Honesty notes
The terminal tool reads only `TERMINAL_ENV`; the bridge fires on the interactive/TUI launch path, so for the non-interactive `-z` harness run `TERMINAL_ENV=docker` (the proven bridged value) was set explicitly.
